# 📘 Документация API Brawl Stats Bot v2

Базовый URL: **`http://130.12.46.224`**

Все защищённые эндпоинты (кроме `POST /generate_key`) требуют передачи API-ключа в заголовке **`X-API-Key`**.  
По умолчанию каждый ключ имеет лимит **10 000 запросов в сутки**. При превышении возвращается статус `429`.

---

## 🔐 Аутентификация

```http
X-API-Key: ваш_ключ
```

---

## 📡 Эндпоинты

### 1. Получение нового ключа  
**`POST /generate_key`**

Публичный эндпоинт для генерации нового API-ключа.

#### Параметры (в теле JSON)

| Поле | Тип | Описание | По умолчанию |
|------|-----|----------|--------------|
| `name` | `string` | Название приложения/клиента | `"Public"` |
| `daily_limit` | `integer` | Лимит запросов в сутки (будет применён, если не превышает максимальный) | `10000` |

#### Пример запроса

```bash
curl -X POST http://130.12.46.224/generate_key \
  -H "Content-Type: application/json" \
  -d '{"name":"MyApp", "daily_limit":10000}'
```

#### Ответ

```json
{
  "key": "eyJhbGciOiJIUzI1NiIs...",
  "name": "MyApp",
  "daily_limit": 10000,
  "created_at": "now"
}
```

### 3. Получение списка игроков  
**`GET /players`**

Возвращает список тегов игроков, хранящихся в базе данных (из GitHub репозитория).

#### Параметры (query)

| Параметр | Тип | Описание | По умолчанию |
|----------|-----|----------|--------------|
| `limit` | `integer` | Максимальное количество записей (≤500) | `100` |
| `offset` | `integer` | Смещение для пагинации | `0` |

#### Пример запроса

```bash
curl -H "X-API-Key: ваш_ключ" "http://130.12.46.224/players?limit=10&offset=0"
```

#### Ответ

```json
{
  "total": 5432,
  "limit": 10,
  "offset": 0,
  "players": [
    {
      "tag": "8UG9C0L",
      "download_url": "https://raw.githubusercontent.com/..."
    },
    ...
  ]
}
```

### 4. Получение данных игрока  
**`GET /player/{tag}`**

Возвращает полную информацию об игроке по его тегу (без символа `#`).

#### Параметры (path)

| Параметр | Тип | Описание |
|----------|-----|----------|
| `tag` | `string` | Тег игрока (например, `8UG9C0L`) |

#### Пример запроса

```bash
curl -H "X-API-Key: ваш_ключ" "http://130.12.46.224/player/8UG9C0L"
```

#### Ответ (пример)

```json
{
  "tag": "#8UG9C0L",
  "name": "Carlos",
  "trophies": 32000,
  "highest_trophies": 35000,
  "exp_level": 150,
  "exp_points": 12800,
  "wins_3v3": 1800,
  "wins_solo": 350,
  "wins_duo": 420,
  "club_tag": "#2YCCU",
  "icon_id": 280000,
  "last_updated": "2025-03-28T10:00:00Z"
}
```

**Поля ответа:**

| Поле | Описание |
|------|----------|
| `tag` | Тег игрока с `#` |
| `name` | Никнейм |
| `trophies` | Текущие трофеи |
| `highest_trophies` | Максимальное количество трофеев |
| `exp_level` | Уровень опыта |
| `exp_points` | Очки опыта |
| `wins_3v3` | Победы в режиме 3×3 |
| `wins_solo` | Победы в соло-шоудауне |
| `wins_duo` | Победы в дуо-шоудауне |
| `club_tag` | Тег клуба (если есть) |
| `icon_id` | ID иконки профиля |
| `last_updated` | Дата последнего обновления |

### 5. История трофеев игрока  
**`GET /player/{tag}/history`**

Возвращает историю изменения трофеев за последние N дней (по умолчанию 30). Данные собираются фоновой задачей каждые 6 часов.

#### Параметры (path и query)

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| `tag` | `string` | Тег игрока | ✅ |
| `days` | `integer` | Количество дней истории (1–365) | ❌ (по умолчанию 30) |

#### Пример запроса

```bash
curl -H "X-API-Key: ваш_ключ" "http://130.12.46.224/player/8UG9C0L/history?days=7"
```

#### Ответ

```json
{
  "player_tag": "#8UG9C0L",
  "history": [
    { "date": "2025-03-28", "trophies": 32000 },
    { "date": "2025-03-27", "trophies": 31850 },
    ...
  ]
}
```

### 14. Поиск игроков по имени  
**`GET /search/players`**

Ищет игроков, чьё имя содержит заданную подстроку (регистронезависимо). Поиск осуществляется через Redis-индекс, который обновляется в фоне каждые 2 часа. При первом запросе после запуска сервера может потребоваться несколько минут на построение индекса, в этом случае ответ будет содержать сообщение `note`.

#### Параметры (query)

| Параметр | Тип | Описание | Обязательный |
|----------|-----|----------|--------------|
| `name` | `string` | Часть имени (минимум 1 символ) | ✅ |

#### Пример

```bash
curl -H "X-API-Key: ваш_ключ" "http://130.12.46.224/search/players?name=Carlos"
```

#### Ответ

```json
{
  "query": "Carlos",
  "results": [
    {
      "tag": "#8UG9C0L",
      "name": "Carlos"
    },
    ...
  ]
}
```

## ⭐ Система рейтинга

Рейтинг поощряет активных пользователей, которые помогают наполнять базу данных и тестировать функционал.  
Очки начисляются **только на сервере** и не могут быть сфальсифицированы локально.

### Как заработать очки

| Действие | Очки | Примечание |
|----------|------|------------|
| Просмотр профиля игрока | 1 | за каждого уникального игрока в день |
| Просмотр боевого лога | 1 | за каждого уникального игрока в день |
| Просмотр клуба | 1 | за каждый уникальный клуб в день |
| Поиск игрока по имени | 5 | за каждый уникальный поисковый запрос |
| Сохранение статистики в PNG | 5 | за каждого уникального игрока |
| Полный сбор данных клуба | 10 | за каждый уникальный клуб |
| Обнаружение командной игры | 10 | за каждую уникальную пару игроков |
| Поиск существующих игроков | 2 | за каждый запуск массового поиска |
| Заполнение базы (однократно) | 20 | за одну операцию |
| Непрерывное заполнение | 20 | за каждого нового игрока, добавленного в базу |
| Генерация кодов | 2 | за один запуск |
| Синхронизация с GitHub | 5 | за одну синхронизацию |

### Защита от накрутки

Чтобы очки начислялись только за реальную активность, введены следующие правила:

- Одно и то же действие с одним объектом (игроком, клубом, поисковым запросом) можно выполнить **не чаще одного раза в 5 минут**. Например, если вы пять раз подряд посмотрите профиль одного и того же игрока, очки начислятся только за первый раз.
- Для операций, не привязанных к объекту (например, генерация кодов или синхронизация), действует тот же принцип – повторный вызов в течение 5 минут не принесёт новых очков.

Эти меры предотвращают искусственный рост рейтинга и сохраняют его объективность.

### Как посмотреть рейтинг

- **В CLI**: выберите пункт меню **⭐ Мой рейтинг**.
- **Через API**: выполните запрос  
  ```bash
  curl -H "X-API-Key: ваш_ключ" http://130.12.46.224/rating/my
  ```

Рейтинг отображается в верхней строке меню и может служить индикатором вашего вклада в развитие проекта.

## 🧠 Примечания по наполнению данных

- **Данные игроков и клубов** загружаются из GitHub-репозитория и доступны сразу.
- **История трофеев и клубов** начинает накапливаться через 6 часов после запуска сервера (фоновый сбор).
- **Бои, карты, командная статистика** требуют интеграции с официальным API Brawl Stars (или другого источника) для наполнения таблицы `battles`. Без этого соответствующие эндпоинты будут возвращать пустые данные.
- **Мастерство бойцов** также требует ключа официального API и будет накапливаться по мере запросов.
- **Поиск по имени** использует Redis-индекс, который строится в первые минуты после старта. До построения индекса поиск может возвращать сообщение о том, что индекс ещё строится.
